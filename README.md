### ⛓️ Github PR 방식

#### ✔️ 브랜치 전략

- Main : 배포했거나 곧 배포할 코드 관리합니다.
- Feature : 기능 개발을 진행할 때 사용합니다. 기능을 완성할 때까지 유지하고 완성 시 Main 브랜치로 merge 합니다.
- Release : 배포를 준비하는 브랜치로 배포를 필요한 메타데이터를 준비합니다.
- Hotfix :  배포한 버전에서 발생한 버그를 수정하는 브랜치입니다.
